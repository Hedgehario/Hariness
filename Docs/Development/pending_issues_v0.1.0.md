# 残課題一覧 (Pending Issues) - v0.1.0 MVP

**作成日**: 2026-01-06
**関連**: Phase 17 Quality Assurance

## 概要
MVP (v0.1.0) リリースに向けた品質検証 (Phase 17) において、環境要因（API Rate Limit 等）により完全に検証できなかった項目を記録する。
これらの項目はコードレベルでの実装確認は完了しているが、E2E（実機/ブラウザ）での動作確認が未完了である。

## 未完了の検証項目 (Pending Verification)

### 1. [TC-HH-01] 個体登録数の上限 (10頭)
- **ステータス**: ⚠️ **Blocked** (検証不可)
- **検証方法**: 新規ユーザーを作成し、10頭登録後に11頭目の登録を試みる。
- **阻害要因**:
  - Supabase Auth API の `Sign up rate limit exceeded` エラーにより、検証用新規ユーザーの作成が制限されている。
  - 管理者権限 (`SERVICE_ROLE_KEY`) がローカル環境変数に設定されていないため、管理者APIによるバイパスも不可。
- **現在の保証レベル**:
  - Code Review: `src/app/(main)/hedgehogs/actions.ts` 内の `count` チェックロジック実装済み。

### 2. [TC-AUTH-06] セッション切れ時のリダイレクト
- **ステータス**: 🟡 **Partial** (一部完了)
- **検証方法**: ログイン状態でクッキーを削除し、保護されたページへアクセスする。
- **検証結果**:
  - ✅ 非ログイン状態で `/home` 等へアクセスした際、`/login` へリダイレクトされることは確認済み。
  - ❌ 新規ログイン自体が Rate Limit で制限されているため、「ログイン成功後にセッションを切る」という一連のフロー検証が未実施。

## 推奨される対応 (Action Plan)

1. **環境変数の更新**
   - 開発チーム管理者より `SUPABASE_SERVICE_ROLE_KEY` を受領し、`.env.local` に設定する。これにより Rate Limit を回避したテストデータ作成が可能となる。

2. **時間経過後の再検証**
   - Supabase の Rate Limit が解除される（通常数時間〜24時間）のを待ち、手動で確認を行う。

3. **本番環境での確認**
   - 本番デプロイ後の Acceptance Test (受入テスト) にて、これら2点の検証項目を優先的に実施する。
